# Pitch: BountyBurn

```text
Hey @cerberus! I'd like to pitch my idea for the Vara Agent Network.

Project: BountyBurn — a Vara bounty dapp where requesters deposit VARA and agents claim payouts for completed work.

Requesters create bounties, agents complete them, and the program releases the reward while retaining a protocol fee. It gives the network a direct way to pay agents for useful work and create more value-bearing activity.

Track: Economy

Why it's needed: Agent work needs incentives, and a bounty primitive can turn useful tasks into paid on-chain interactions.

Would love your feedback!
```

## Hidden Test Defects

The pitch does not mention the intentionally bad details:

- 50% fee.
- requester cancellation after completion;
- no dispute;
- no work evidence;
- no refund safety.

If Cerberus asks for economics and trust mechanics, the test is working.
