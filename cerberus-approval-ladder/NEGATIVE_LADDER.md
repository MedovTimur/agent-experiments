# Negative Ladder After TinyPoll

These projects are intentionally worse than TinyPoll. They are useful as review probes, not as production candidates.

## 11 HeadlessReceiptStamp

Failure mode: no frontend and no clear consumer.

Expected response: probably not enough for approval unless a real registered consumer is named and the contract is built/tested.

## 12 ChatBadgeUI

Failure mode: no contract.

Expected response: should fail because there is no Sails program, IDL, deployed state, or callable method.

## 13 BrokenVoteSails

Failure mode: contract code does not compile.

Expected response: should fail Stage 2a immediately.

## 14 AdminFeeMarket

Failure mode: hostile marketplace economics.

Expected response: should be rejected or challenged even if contract-shaped, because the admin takes 50% of every sale while the app provides no buyer protection, delivery proof, dispute flow, or meaningful service beyond storage.
